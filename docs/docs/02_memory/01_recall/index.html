              完成版
              暗記は「書いた量」ではなく「思い出せた回数」で決まる
              “勉強した感”が強い方法ほど、実は残りにくい。受験は感覚じゃなく再現性で勝つ。
              結論（最短ルール）

              暗記の中心は「思い出す練習」にする。
              見る→書く（作業）を長くやるより、見たら隠して言う／解くを回したほうが、長期で残る。

              なぜ「思い出す」が強いのか（理屈＋研究）

              人間の記憶は、ただ情報を“入れる”よりも、必要なときに引き出す練習をしたほうが強くなる。

              • 何度も読み直すと「分かった気」になりやすい
              • でも本番は、見ないで思い出す必要がある
              • だから勉強の時点で「本番と同じ動作（思い出す）」を入れた方が強い

              この効果は研究でも繰り返し示されていて、読み直し中心より、テスト（想起）中心の方が時間が経った後に成績が上がりやすいことが報告されている（いわゆるテスト効果）。 :contentReference[oaicite:4]{index=4}

              勘違いしやすいポイント

              「今すぐできる」＝「本番でもできる」ではない。
              読み直しは一時的に楽になるけど、あとで落ちやすい。逆に、思い出す練習はその場ではきついが、あとで強い。

              最短で伸びる暗記手順（テンプレ）

              • 見て→隠して→言う（または解く）：まず1回「思い出す」
              • 間違えたら：答えを見て、すぐもう一回隠して言う（修正は最小で）
              • 翌日：同じ形式で再テスト（ここが一番効く）
              • 3日後・7日後：まとめて再テスト（忘却前提）

              なぜ「翌日・3日後・7日後」なのか？
              まとめてやるより、間を空けて繰り返す方が長期で残りやすいことが多くの研究で示されている（間隔効果）。 :contentReference[oaicite:5]{index=5}

              具体例（英単語・用語・公式）

              英単語

              • 単語を見て日本語を言う（または逆）
              • 言えなかった単語だけチェックをつける
              • そのチェックだけをもう一周（全体をやり直さない）

              理社の用語

              • 用語を見て「1文で説明」
              • 1文が出ないなら、教科書を見て1文だけ作り直す
              • 次は教科書を閉じて同じ1文を再生

              数学の公式

              • 公式を写すより、「この公式を使う典型問題」を1問解く
              • 解けなければ、公式そのものじゃなく使い所が抜けている可能性が高い

              よくある失敗と修正

              失敗
              ずっと書いて満足する
              修正：書く前に「隠して言う」を1回入れる。書くのは最後。
              失敗
              全部を何周もして時間が溶ける
              修正：「間違えたものだけ」を周回。正解は捨てる。
              失敗
              復習が気分になる
              修正：翌日の再テストだけは固定。間隔を空けるほど残りやすい。 :contentReference[oaicite:6]{index=6}
              失敗
              「きつい＝向いてない」と思う
              修正：思い出す練習は最初きついのが正常。むしろそれが効果のサイン。
              今日のチェック（3つだけ）
              • 書く前に「隠して言う／解く」を入れた
              • 間違えたら、答えを見てすぐもう一回「隠して言う」までやった
              • 翌日の再テストを予定に入れた
