              完成版
              復習は「気分」じゃなく「予定」にする
              人は忘れる。忘れるのが普通。だから、忘れる前提で“復習の出番”を先に決める。
              結論（これだけ）

              復習の強さは「回数」より「タイミング」で決まる。
              いちばん大事なのは 翌日。次が 3日後。その次が 7日後。
              まずは当日＋翌日だけ固定すれば、成績は動く。

              なぜ「翌日」が最重要なのか（理屈）

              暗記は「入れたら終わり」じゃない。
              時間が経つと、思い出すのに必要な“道”が細くなる。

              • 翌日は、まだ道が完全に消えていない
              • でも、そのまま放置すると一気に戻れなくなる
              • だから翌日にもう一回“思い出す”と、道が太くなる

              ここを外すと、復習は「やった気」になりやすい。

              復習の基本形（最短テンプレ）

              • 当日：間違いの原因だけ直す（長くやらない）
              • 翌日：同じ形式で再テスト（ここが本番）
              • 3日後：もう一回テスト（“落ちる所”を確定させる）
              • 7日後：まとめテスト（長期に残す）
              重要：復習は「読み直し」じゃなく「再テスト」

              復習の目的は納得じゃない。次に正解すること。
              だから復習は「もう一回解く／言う」が中心。

              どれくらい復習すればいい？（時間の目安）

              「復習が長すぎて続かない」が一番もったいない。
              目安はこれでOK。

              当日（最短）
              3〜10分
              原因ラベル＋1問やり直し。これで十分。
              翌日（最重要）
              10〜20分
              再テスト中心。できない所だけ絞る。

              実例：英単語の復習設計（そのまま使える）

              • 当日：30語テスト → 間違いだけチェック
              • 翌日：チェック語だけテスト（できたら外す）
              • 3日後：残ったチェックだけテスト
              • 7日後：当日30語をもう一回テスト（抜けの回収）

              ポイントはこれ：

              • 全部を何周もしない
              • 間違いだけを追いかける
              • 翌日だけは必ずやる

              実例：数学の復習設計（「解き方」が残る形）

              数学は暗記というより「手順」を残す復習になる。

              • 当日：ミスの種類を決める（知識/理解/手順/読み/ケアレス）
              • 当日：解き方を“3手”で書く（長文禁止）
              • 翌日：同類題を1問解く（見ずに）
              • 3日後：同類題をもう1問（時間を測る）

              よくある失敗 → 直し方

              失敗
              復習が「読み直し」になる
              直し方：最初に必ず「隠して言う／解く」を入れる。
              失敗
              復習の予定がなく、気分になる
              直し方：翌日だけは固定。カレンダーでも紙でもOK。
              失敗
              全部を何度もやって時間が溶ける
              直し方：「間違いだけ」を回す。正解は捨てる。
              失敗
              復習が長すぎて続かない
              直し方：当日は3分でもOK。翌日の再テストを守る。
