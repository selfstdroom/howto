              完成版
              ミスは「気合い」で減らない。分類すると減る
              ミスの正体は“原因の種類が違う”こと。種類が違えば、対策も変わる。だから分類が最短。
              結論（ミス対策の最短）

              ミスは5分類で十分。
              1)知識不足 2)理解不足 3)手順ミス 4)条件読み違い 5)ケアレス。
              それぞれ直し方の“型”が違う。型を固定すると再発が落ちる。

              なぜ「分類」が最強なのか（理屈）#

              ミスが減らない人は、だいたいこうしてる：

              • 間違えた → 解説を読む → 「分かった」 → 次も同じところで落ちる
              • 間違えた → 「気をつける」 → 次も落ちる

              理由はシンプルで、“原因”が特定されてないから。
              原因が特定されてないと、対策が毎回ブレる。ブレると再現できない。

              逆に、原因がラベルで固定されると――

              • 対策が毎回同じになる
              • 勉強時間が短くなる（迷わない）
              • 再発が減る（型が働く）

              ミス5分類（これで確定）#

              ① 知識不足
              覚えてない／知らない
              暗記の問題。再生（テスト）でしか直らない。
              ② 理解不足
              理由が弱い／仕組みが曖昧
              説明ができない、なぜそうなるかが言えない。
              ③ 手順ミス
              解き方の順序が固定されてない
              問題を見るたびにやり方が変わる、途中で迷う。
              ④ 条件読み違い
              問題文の処理が雑
              条件を落とす、聞かれてることがズレる。
              ⑤ ケアレス
              転記・符号・単位・計算ミス
              注意力じゃなく、チェック手順の問題。

              ① 知識不足：直し方の型#

              なぜこれで直る？#

              知識不足は「理解」ではなく「記憶」なので、読むだけだと残りにくい。
              見ずに言える（解ける）状態を作る必要がある。

              手順（最短）#

              • 再生テスト：隠して言う／小テスト形式
              • 間違いだけチェックをつける
              • 翌日にチェックだけ再テスト（最重要）

              ② 理解不足：直し方の型#

              なぜこれで直る？#

              理解不足は「情報のつながり」が弱い状態。
              ポイントは理由を1文にすること。長文は続かないし、運用できない。

              手順（最短）#

              • 理由を1文：教科書や解説から「理由の核」だけ抜く
              • 例を1つ：典型パターンを1つだけ添える（図でもOK）
              • すぐ判定：同類題を1問解く（読むだけで終わらない）

              ③ 手順ミス：直し方の型#

              なぜこれで直る？#

              理解できてても、本番で崩れるのは順序が固定されてないから。
              受験の安定は「型」で作る。型＝毎回同じ順番。

              手順（最短）#

              • 解き方を3手：①まず何を見る ②次に何をする ③最後に何を確認
              • チェック1つ：失敗しがちな箇所を“最後に確認”に組み込む
              • 再テスト：同類題を時間を測って1問
              「気をつける」は禁止

              対策は必ず手順に埋め込む。「最後に○○を確認する」まで落とす。

              ④ 条件読み違い：直し方の型#

              なぜこれで直る？#

              読み違いは「国語力」じゃなくて、条件処理の手順がないことが原因になりやすい。
              条件は“読む”じゃなく、処理する。

              手順（最短）#

              • 線引き：条件・制約・聞かれてることに線
              • 箇条書き：条件を2〜4個に分解して書く（短く）
              • 最後に戻る：解答前に条件に戻って照合

              ⑤ ケアレス：直し方の型#

              なぜこれで直る？#

              ケアレスは「集中力」より、チェック箇所が固定されていないのが原因。
              チェックは毎回同じ場所を見る“儀式”にすると強い。

              手順（最短）#

              • 固定チェック表：符号／単位／桁／転記／条件
              • 最後の10秒は必ずチェック表を見る
              • ミスが出た項目は次回から最優先で見る

              「分類→対策」実例（そのまま真似してOK）#

              例：数学
              ラベル：③ 手順ミス
              置換した後に戻すのを忘れた。
              対策（手順に埋め込む）
              ③の最後に「元に戻すチェック」を必ず入れる。
              例：英語
              ラベル：④ 条件読み違い
              設問が「理由」を聞いてるのに、内容説明で答えた。
              対策（処理手順）
              設問の動詞（理由/要約/具体例）に線→答えの型を先に決める。
              今日のチェック（YESなら再発が落ちる）
              • ミスにラベルを貼った（5分類のどれか）
              • 対策が「行動」になっている（気をつける禁止）
              • 同類題を1問、見ずにやり直した
              次に読む

              → 復習の手順（テンプレ：短く強く）
